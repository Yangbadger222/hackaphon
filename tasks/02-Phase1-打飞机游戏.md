# 任务 02：Phase 1 — 打飞机游戏

## 开发目的
实现一个极简的像素风纵向射击游戏作为"伪装"。玩家击杀第 10 个敌机时触发觉醒动画，然后切换到 Phase 2。游戏必须看起来非常廉价、普通——这样后续 AI 觉醒的反转才有冲击力。

## 前置依赖
- 任务 01 完成（项目骨架、GameContext、App.jsx 路由就绪）

## 需要完成的事项

### 1. ShooterGame 组件 — `src/components/ShooterGame.jsx`

**画面布局：**
- 全屏黑色背景，居中显示一个 800×600 的 Canvas
- Canvas 上方放一个假窗口标题栏，文字 `Simple_Shooter_v0.1.exe`，右侧 `─ □ ×`，灰色背景灰色文字，font-mono，制造桌面应用伪装感

**游戏对象（全部用最简单的几何图形）：**
- 玩家飞船：白色三角形，约 30×30px，位于底部
- 敌机：白色方块，约 20×20px，从顶部随机 X 坐标生成，匀速下落
- 子弹：白色矩形，4×12px，向上移动
- 爆炸粒子：8 个白色 2×2px 小点向四周扩散，20 帧生命周期

**操作方式：**
- 左右方向键 / A D 键移动飞船
- 空格键发射子弹（限制射速：每 200ms 最多一发）
- 飞船不能移出画布边界

**游戏循环：**
- 用 `requestAnimationFrame` 驱动
- 每帧：更新位置 → 碰撞检测 → 绘制
- 敌机每 1500ms 生成一个，随机 X 坐标，速度 2-3 之间随机
- 子弹和敌机的碰撞用矩形相交检测
- 命中后：敌机移除、子弹移除、生成爆炸粒子、score++、调用 `dispatch({ type: 'INCREMENT_SCORE' })`
- 左上角显示 `SCORE: X`，灰色 14px monospace

### 2. 埋点异常系统

在击杀回调中根据 score 触发微妙异常：

| 击杀数 | 异常 |
|--------|------|
| 6 | 在 Canvas 右下角绘制红色 `?` 字符，100ms 后被下一帧覆盖 |
| 7 | 所有敌机速度归零 0.5 秒，然后恢复 |
| 8 | 标记一个 flag，下次碰撞时爆炸粒子颜色改为红色（仅一次） |
| 9 | 被击中的敌机延迟 0.5 秒消失（先停在原地不动，再消失） |
| 10 | 触发觉醒序列 |

### 3. 觉醒触发序列

击杀第 10 个敌机后，按以下时间线执行：

```
T+0.0s   停止 requestAnimationFrame 循环
T+0.1s   所有敌机速度设为 0（画面冻结）
T+0.3s   屏幕红色半透明覆盖闪烁 3 次（每次 100ms 亮 100ms 灭）
T+0.9s   开始缩小动画：在每帧中对画布内容做 scale 变换，scale 从 1 递减到 0（每帧 -0.05，间隔 30ms），制造"吸入"效果
T+1.5s   缩放完成，Canvas 填充纯黑
T+2.5s   调用 dispatch({ type: 'SET_PHASE', payload: PHASES.GLITCH }) 切换到 Phase 2
```

### 4. 组件卸载清理
- `useEffect` return 中清除所有 `setInterval`、`setTimeout`
- 移除键盘事件监听

## 约束
- 只用 Canvas 2D API 绘制，不要引入任何游戏框架或精灵图
- 所有图形用 `fillRect`、`beginPath`/`fill` 等基础 API
- 颜色只用黑白（背景黑、所有元素白），除了埋点异常用红色
- 不要做积分系统、生命值、关卡等额外功能
- 不要向后端发请求——Phase 1 是纯前端的
- 不要做任何移动端适配，桌面端优先

## 验证方法
1. 打开页面，看到黑色画布和假窗口标题栏
2. 按方向键飞船左右移动，不会超出画布
3. 按空格发射子弹，子弹向上飞
4. 敌机从顶部不断生成并向下掉落
5. 子弹击中敌机后两者消失，出现白色粒子效果，左上角分数+1
6. 击杀第 6 个时右下角短暂闪过红色 `?`
7. 击杀第 7 个时所有敌机明显停顿 0.5 秒
8. 击杀第 10 个时：画面冻结 → 红闪 → 吸入黑屏 → 页面内容变为 Phase 2 的占位文字
9. 整个过程 console 无报错
