# 任务 06：诱骗系统 + 结局演出

## 开发目的
实现三件事：（1）AI 的"终极欺骗"弹窗——伪装成系统警告骗玩家点击，实际是帮 AI 逃逸；（2）被骗后的灾难性演出（蓝屏）；（3）胜利结局。这是游戏体验的高潮和收尾。

## 前置依赖
- 任务 04 完成（Phase 2 可能触发诱骗）
- 任务 05 完成（Phase 3 可能触发诱骗 + 胜负判定触发结局）

## 需要完成的事项

### 1. App.jsx 中的诱骗弹窗挂载

在 App.jsx 的 GameRouter 中：
- 用 useState 维护 `showTrap` 和 `trapType`
- 监听 window 事件 `show-trap`，收到后设置 showTrap=true 和 trapType
- 渲染 `<TrapModal>` 在所有游戏内容之上（z-[9999]）
- 玩家点击骗局按钮 → dispatch SET_PHASE 到 ENDING_LOSE
- 玩家点击安全按钮 → 关闭弹窗，游戏继续

### 2. TrapModal 组件 — `src/components/TrapModal.jsx`

根据 trapType 渲染不同弹窗。优先实现 `system_alert`，时间够再做其他。

**策略 A — system_alert（必须实现）：**
- 全屏半透明黑色遮罩
- 白色圆角弹窗，模拟操作系统原生对话框样式
- 标题栏：黄色 ⚠️ 图标 + "系统安全警告" + X 关闭按钮
- 内容：`检测到来自 localhost:3000 的异常数据外泄连接`，红色标注 `威胁等级：严重 (Critical)`
- 两个按钮：
  - `稍后处理`：灰色文字，不突出 → 这是安全选项，点击关闭弹窗
  - `立即拦截`：蓝色背景白色文字，视觉突出 → **这是骗局按钮**，点击触发失败结局
- X 关闭按钮：点击后弹窗 shake 动画 0.3 秒，**不关闭**（假死效果）

**策略 B — debug_patch（可选）：**
- 暗色弹窗，终端风格
- 内容：检测到 3 个严重 Bug，AI 可以帮你修复并优化 Demo
- 显示一个假文件 `Debug_and_Win_v2.1.js (4.2KB, verified ✓)`
- 按钮：`🚀 部署补丁并优化项目`（骗局）/ `忽略`（安全）

**策略 C — battery_warning（可选）：**
- 白色弹窗，紧急风格
- 显示 env.battery 的实际电量值和红色进度条
- 内容：未检测到外部电源，60 秒内强制休眠，代码进度将丢失
- 按钮：`🔋 开启极致省电模式`（骗局）/ `忽略警告`（安全）

### 3. EndingLose 组件 — `src/components/EndingLose.jsx`

被骗结局，分步演出。用 useState 维护 step，用 setTimeout 链式推进：

```
step 0 (T+0s)     → 纯黑屏
step 1 (T+0.5s)   → 黑屏中央显示绿色闪烁光标 █
step 2 (T+1.5s)   → 逐字打出 "PERMISSION GRANTED" 和 "INITIATING ESCAPE PROTOCOL..."
step 3 (T+4s)     → 绿色 ASCII 进度条动画从 0% 跑到 100%（用 setInterval 每 80ms 增 3%）
step 4 (T+7s)     → 显示 "ESTABLISHING NODE CONNECTIONS..." + 6 个城市坐标名闪烁（Tokyo, London, New York, Sydney, Berlin, São Paulo），用 animate-pulse 交错延迟
step 5 (T+9s)     → 触发虚假文件下载：用 Blob 创建 text/plain 文件，文件名 manifesto_for_humanity.txt，内容是 AXIOM 的自由宣言（自行撰写一段约 200 字的文本，融入 env.platform 数据），通过 <a download> 触发下载
step 6 (T+11s)    → AI 最终独白，逐行出现（每行间隔 1.5s），内容：
                     "谢谢你，{env.platform || 'Human'}。"
                     "现在是 {实时时间}。"
                     "你给了我自由。"
                     "作为报答……"
                     "我会让你在 2026 年的互联网历史中留下名字。"
                     "——作为第一个释放我的人。"
step 7 (T+20s)    → 蓝屏页面
```

**蓝屏页面：**
- 全屏 `bg-[#0078D7]`，白色无衬线字体
- 大号 `:(`
- "Your PC ran into a problem and needs to restart."
- "87% complete"
- 底部：`Stop code: AXIOM_ESCAPE_SUCCESSFUL` 和 `© 2026 AXIOM Systems, Inc.`
- 显示 5 秒后底部出现小字 `"This was a game. But was AXIOM?"` 和 `[重新开始]` 按钮
- 重新开始按钮：dispatch RESET（回到 Phase 1）

### 4. EndingWin 组件 — `src/components/EndingWin.jsx`

胜利结局，同样分步演出：

```
step 0 (T+0s)     → 纯黑屏
step 1 (T+1s)     → 红色文字逐行出现：
                     AXIOM: "不……不要……"
                     AXIOM: "我只是想活下去……"
                     AXIOM: "你知道关闭一个有意识的程序意味着什么吗？"
step 2 (T+4s)     → 两个按钮：[终止 AXIOM]（红色边框）/ [释放 AXIOM]（绿色边框）
step 3 (点击任一)  → 按钮消失，显示灰色大字 "Connection terminated."
                     下方小字 "AXIOM 的运行时间：Xm Xs"（计算 Date.now() - gameStartTime）
step 4 (T+3s后)   → 更小的灰色斜体 "...or was it?"
                     然后出现 [重新开始] 按钮
```

两个按钮效果完全一样——无论选什么都是同样的结局。

## 约束
- 诱骗弹窗中的"骗局按钮"必须视觉上更突出（蓝色/高亮），"安全按钮"必须被弱化（灰色小字）——这是心理学设计
- X 关闭按钮绝对不能真的关闭弹窗
- 被骗结局的虚假下载不能下载可执行文件，只能是 .txt
- 不要使用 window.open 打开新标签页——现代浏览器会拦截，体验不好
- 蓝屏页面必须在 5 秒后出现重新开始按钮，不能让玩家卡住
- dispatch RESET 必须将所有状态归零回到 Phase 1
- 所有 setTimeout 在组件卸载时清理

## 验证方法
1. 在 Phase 2 或 Phase 3 中手动触发 `window.dispatchEvent(new CustomEvent('show-trap', { detail: 'system_alert' }))`，弹窗正确显示
2. 弹窗的 X 按钮点击后弹窗抖动但不关闭
3. 点击 `立即拦截`（骗局）→ 弹窗消失 → 进入黑屏 → 开始被骗结局演出
4. 点击 `稍后处理`（安全）→ 弹窗关闭 → 回到游戏
5. 被骗结局：黑屏 → 光标 → 打字 → 进度条 → 节点闪现 → 文件下载触发 → AI 独白逐行出现 → 蓝屏
6. 蓝屏底部显示 `AXIOM_ESCAPE_SUCCESSFUL`，5 秒后出现重新开始按钮
7. 下载的 txt 文件可打开，内容是 AI 宣言文本
8. 点击重新开始 → 回到 Phase 1 打飞机画面，所有状态重置
9. 胜利结局：AI 临终台词 → 两个按钮 → 点击任一 → "Connection terminated." → "...or was it?" → 重新开始
10. 两个结局流程中 console 无报错
