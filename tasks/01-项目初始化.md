# 任务 01：项目初始化

## 开发目的
搭建前后端项目骨架，使后续任务可以直接在此基础上开发组件和 API。本任务不实现任何游戏逻辑，只建立基础设施。

## 需要完成的事项

### 1. 前端项目
- 用 Vite + React 创建项目
- 安装并配置 Tailwind CSS（使用 `@tailwindcss/vite` 插件）
- 在 `src/index.css` 中添加 `@import "tailwindcss";`

### 2. 后端项目
- 在项目根目录创建 `server/` 文件夹
- 初始化 Node.js 项目，安装 `express`、`cors`、`dotenv`
- 安装 LLM SDK（`openai` 或 `@anthropic-ai/sdk`，看用户选择）
- 创建 `server/index.js`：Express 服务器，监听 3001 端口，启用 CORS 和 JSON 解析
- 创建 `server/.env` 存放 API Key（添加占位符，提示用户填写）
- `/api/game` 端点先返回 `{ message: "API ready" }` 占位

### 3. 全局状态管理 — `src/context/GameContext.jsx`
- 使用 `createContext` + `useReducer` 模式
- 状态结构和 PHASES 枚举见 `00-总览` 文档
- 导出 `GameProvider`、`useGame()`、`PHASES`
- reducer 需处理的 action type：`SET_PHASE`、`INCREMENT_SCORE`、`ADD_CHOICE`、`SET_ENV`、`SET_CSS_ATTACK`、`SET_INTEGRITY`、`SET_ESCAPE_PROGRESS`、`ADD_DIALOGUE`、`RESET`
- `SET_INTEGRITY` 和 `SET_ESCAPE_PROGRESS` 需要 clamp 到 0-100 范围

### 4. 环境感知 Hook — `src/hooks/useEnvSensor.js`
- 在 `useEffect` 中一次性采集以下数据，存入 state 返回：
  - `navigator.getBattery()` → battery, charging（try-catch，不支持时设 null）
  - `new Date()` → localTime, timezone, hour
  - `window.screen` → screenWidth, screenHeight
  - `navigator` → userAgent, language, platform
  - `navigator.connection?.effectiveType` → connectionType

### 5. 打字机 Hook — `src/hooks/useTypewriter.js`
- 参数：`(text, speed = 40, startImmediately = true)`
- 返回 `{ displayed, isComplete }`
- 用 `setInterval` 逐字截取 text，完成后 `setIsComplete(true)`
- text 或 speed 变化时重新开始

### 6. API 通信 — `src/api/gameApi.js`
- 导出 `sendGameState(gameState)` 函数
- 向 `${VITE_API_URL || 'http://localhost:3001'}/api/game` 发 POST 请求
- 返回 JSON 响应

### 7. CSS 攻击样式 — `src/styles/attacks.css`
- 定义以下 class：
  - `.attack-blur` → `filter: blur(3px)` + 0.5s transition
  - `.attack-shake` → 使用 `@keyframes shake` 左右抖动，0.15s infinite
  - `.attack-invert` → `filter: invert(1)` + 1s transition
  - `.attack-rotate` → `transform: rotate(180deg)` + 3s transition
  - `.attack-glitch` → 使用 `@keyframes glitch` clip-path 错位
  - `.attack-red-flash` → 红色半透明背景闪烁 3 次

### 8. App.jsx — 阶段路由
- 用 `GameProvider` 包裹
- 根据 `state.phase` 渲染不同组件（ShooterGame / AIDialogue / FirewallUI / EndingLose / EndingWin）
- 将 `state.cssAttack` 作为 className 动态应用到外层容器：`attack-${state.cssAttack}`
- 所有子组件先创建为占位组件（`export default function X() { return <div>TODO</div>; }`）

### 9. 确保 `server/package.json` 中 type 为 module（使用 ES module import）

## 约束
- 不要实现任何游戏逻辑或 UI 设计，只搭骨架和基础设施
- 不要安装未提到的依赖包
- `.env` 文件不要写入真实 API Key，只写占位符并注释说明
- GameContext 的 reducer 不要预设复杂的业务逻辑，保持简单的 state 更新
- CSS 动画不要使用过于复杂的效果，保持性能

## 验证方法
1. 运行 `npm run dev`，浏览器打开后页面显示 ShooterGame 的占位文字，无报错
2. 运行 `node server/index.js`，终端显示 `Server running on 3001`
3. 执行 `curl -X POST http://localhost:3001/api/game -H "Content-Type: application/json" -d '{"phase":1}'`，返回 `{"message":"API ready"}`
4. 在浏览器控制台检查：页面上有一个 `text-green-400` 的元素确实显示绿色（Tailwind 生效）
5. 在任意组件中调用 `useGame()`，能读到 `state.phase === 1`
6. 在任意组件中调用 `useEnvSensor()`，控制台能打印出时间和屏幕信息
7. 手动给 body 添加 `attack-blur` class，页面变模糊
