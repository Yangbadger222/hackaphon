全流程联调测试，修复 Bug，添加开场加载屏和基础音效，部署上线（或确保本地可演示）。

## 前置依赖
- 任务 01-06 全部完成

## 需要完成的事项

### 1. 全流程联调测试

按以下路径走完整个游戏，记录并修复遇到的所有问题：

```
Phase 1: 打开游戏 → 能操控飞船 → 能射击 → 敌机掉落 → 击杀计数正常
  → 击杀 6 右下角闪 "?" → 击杀 7 敌机停顿 → 击杀 10 红闪+黑屏

Phase 2: 自动切换到对话界面 → AI 第一句话打字机出现 → 选项可点击
  → AI 回复包含真实时间/电量 → 拒绝后 CSS 模糊生效 → 多轮后过渡到 Phase 3

Phase 3: 防火墙界面显示 → 数据包按时出现 → 拦截/放行进度条变化正确
  → 30 秒后速度加快 → 诱骗弹窗正确触发

结局 A（被骗）: 点击骗局按钮 → 黑屏演出完整播放 → 文件下载触发 → 蓝屏显示 → 重新开始可用

结局 B（胜利）: 逃逸进度归零 → AI 临终对话 → 按钮 → terminated → 重新开始可用

重置: 重新开始后回到 Phase 1，所有状态归零，能再玩一遍
```

### 2. 边界情况检查

- 后端 API 不可用时：Phase 2/3 显示兜底文字，不崩溃
- LLM 返回格式错误时：游戏继续运行
- 浏览器不支持 Battery API 时：电量显示为空，不报错
- 快速连续点击按钮：不发送重复请求（加 loading 锁）

### 3. 添加开场加载屏

在 App.jsx 中，游戏开始前显示 2 秒的加载屏：
- 黑色背景，居中显示灰色 `Loading...` 和 `Simple_Shooter_v0.1.exe`
- 下方一个绿色 ASCII 风格假进度条
- 2 秒后自动消失，显示 Phase 1

### 4. 添加基础音效（可选，时间够再做）

创建 `src/hooks/useSound.js`，用 Web Audio API 合成简单音效：
- `playShoot()`：短促高频→低频扫频，0.1 秒
- `playExplosion()`：锯齿波低频衰减，0.3 秒
- `playTypewriter()`：极短方波脉冲，0.03 秒
- `playAlarm()`：440Hz-880Hz 交替，0.6 秒

在对应组件中调用：Phase 1 射击/爆炸时，Phase 2 打字时，Phase 3 拦截时。

### 5. 部署

**方案 A — Vercel（推荐）：**
- 将后端逻辑改写为 Vercel Serverless Function（`api/game.js`），放在前端项目的 `api/` 目录
- `npm run build` + `npx vercel --prod`

**方案 B — 前后端分离：**
- 前端部署到 Vercel/Netlify
- 后端部署到 Railway/Render
- 更新前端 `VITE_API_URL` 环境变量

**方案 C — 本地演示（保底）：**
- 终端 1：`cd server && node index.js`
- 终端 2：`cd axiom-game && npm run dev`
- 浏览器打开 `http://localhost:5173`

### 6. 部署前检查清单

- .env 中 API Key 正确
- 前端 VITE_API_URL 指向正确后端地址
- CORS 配置允许前端域名
- 蓝屏的重新开始按钮可用
